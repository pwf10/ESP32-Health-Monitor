# ESP32-Health-Monitor
基于 ESP32 的多模态生理参数监测系统的设计与实现

---

## 项目介绍
本项目是面向个人健康监测的嵌入式物联网系统，以ESP32为主控核心，实现了人体生理参数的采集、传输、可视化与预警功能。系统分为硬件采集端、后端服务、前端可视化三部分，可实时监测心率、血氧、体温等多模态生理数据，支持移动端查看与异常预警，满足日常健康监测的需求。

本项目为本科毕业设计完整源码，包含全部开发文档与可直接运行的代码。

---

## 项目目录结构
ESP32-Health-Monitor/
├── HBuilderProjects/HealthMonitor # 前端部分（移动端 UniApp 项目）
├── health-monitor-server # 服务器部分（Node.js 后端服务）
└── iot # 硬件部分（ESP32 嵌入式代码）


---

## 功能模块
1.  **生理参数采集**：基于ESP32主控，配合传感器实现心率、血氧、体温等参数的实时采集
2.  **数据无线传输**：通过WiFi实现硬件端与后端服务的双向数据通信
3.  **数据存储与处理**：后端服务实现数据的接收、解析、存储与异常判断
4.  **移动端可视化**：UniApp开发的跨平台移动端，实现数据实时展示、历史曲线查看、异常预警
5.  **异常预警功能**：当生理参数超出正常阈值时，自动触发预警提示

---

## 技术栈
- **硬件端**：ESP32微控制器、MicroPython开发、传感器驱动开发
- **后端服务**：Node.js、Express框架、MySQL数据库
- **前端移动端**：UniApp、Vue.js、uView UI组件库、ECharts数据可视化

## 运行说明
### 1. 硬件端部署
1.  给ESP32烧录MicroPython固件
2.  按硬件原理图连接传感器与ESP32主控
3.  修改`iot`目录下的代码，配置WiFi名称、密码、后端服务地址
4.  将代码上传到ESP32，上电运行

### 2. 后端服务部署
1.  进入`health-monitor-server`目录
2.  执行`npm install`安装依赖
3.  修改配置文件，配置MySQL数据库连接信息
4.  执行`npm run start`启动服务

### 3. 前端项目运行
1.  用HBuilderX打开`HBuilderProjects/HealthMonitor`项目
2.  修改配置文件，配置后端服务接口地址
3.  选择运行到浏览器/真机，即可查看项目
